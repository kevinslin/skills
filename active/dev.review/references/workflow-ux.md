# UX Review Workflow

Use this workflow to review whether an experience helps the target user complete real workflows.

## Steps

1. Check task fit.
   - Verify that the primary user goal is clear and the interface supports the expected workflow.
2. Check information architecture.
   - Identify unclear hierarchy, missing context, overloaded labels, and confusing navigation.
3. Check interaction design.
   - Review controls, affordances, feedback, loading states, empty states, errors, and recovery paths.
4. Check accessibility and readability.
   - Review keyboard access, focus, contrast, text sizing, hit targets, and screen-reader clarity when applicable.
5. Check visual discipline.
   - Call out clutter, inconsistent spacing, distracting styling, weak alignment, and text overflow.
6. Look for simplification.
   - Propose concrete removals, regrouping, default choices, or lower-friction flows.

## Severity Guidance

Treat blocked core workflows, inaccessible controls, and unclear destructive actions as high-severity findings.

Clearly label severity for each issue: blocker, major, minor, or nit.
